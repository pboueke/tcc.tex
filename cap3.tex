\section{Arquitetura}

\paragraph{}O projeto desenvolvido se trata de uma aplicação \textit{Django} que conta com duas interfaces, uma para execução de um modelo preditivo em ocorrências controladas e outra para monitoramento do banco de dados, chamadas, respectivamente, de \textit{Showroom} e \textit{Monitor}. Essas duas interfaces são controladas pela aplicação por meio de duas classes de mesmo nome, vistas na figura 3.1.

\subsection{Showroom}

\paragraph{}A \textit{view Showroom} é caracterizada pela existência de casos sobre os quais serão inferidas classificações a partir do modelo de aprendizado de máquina carregado na aplicação. Toda execução gerada a partir dessa \textit{view} segue os passos vistos no diagrama de sequência apresentado na figura 3.2. Os dados para a predição são coletados, o evento é registrado na tabela de log eventos, o resultado é computado e a \textit{view} é atualizada com os resultados da predição.
